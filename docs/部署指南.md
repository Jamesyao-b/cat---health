# 喵健康 - 部署指南

## 一、部署概述

本项目采用前后端分离架构，推荐使用以下免费平台部署：

| 服务 | 推荐平台 | 免费额度 |
|-----|---------|---------|
| 后端 | Railway | 每月 $5 免费额度 |
| 前端 | Vercel | 无限制 |
| 数据库 | Railway PostgreSQL | 包含在后端额度中 |

---

## 二、后端部署

### 方式一：Railway 部署（推荐）

#### 步骤 1：注册 Railway
1. 访问 https://railway.app
2. 使用 GitHub 账号登录

#### 步骤 2：创建项目
1. 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择你的仓库
4. 选择 `server` 目录作为根目录

#### 步骤 3：配置环境变量
在 Railway 项目设置中添加：
```
ZHIPU_API_KEY=你的智谱API密钥
```

#### 步骤 4：获取后端地址
部署完成后，Railway 会提供一个地址，如：
```
https://miao-health-server.railway.app
```

---

### 方式二：Render 部署

#### 步骤 1：注册 Render
1. 访问 https://render.com
2. 使用 GitHub 账号登录

#### 步骤 2：创建 Web Service
1. 点击 "New" → "Web Service"
2. 连接 GitHub 仓库
3. 设置：
   - Root Directory: `server`
   - Build Command: `npm install`
   - Start Command: `npm start`

---

## 三、前端部署

### Vercel 部署（推荐）

#### 步骤 1：注册 Vercel
1. 访问 https://vercel.com
2. 使用 GitHub 账号登录

#### 步骤 2：导入项目
1. 点击 "New Project"
2. 导入你的 GitHub 仓库
3. 设置：
   - Root Directory: `client`
   - Framework Preset: Vite

#### 步骤 3：配置环境变量
添加后端 API 地址：
```
VITE_API_URL=https://你的后端地址
```

#### 步骤 4：部署
点击 Deploy，等待部署完成。

---

## 四、修改前端 API 地址

部署前，需要修改前端的 API 地址：

### 方法 1：使用环境变量

创建 `client/.env.production`：
```
VITE_API_URL=https://你的后端地址
```

修改 `client/src/App.jsx` 和其他使用 API 的文件：
```javascript
const API_BASE = import.meta.env.VITE_API_URL || '/api'
```

### 方法 2：修改 vite.config.js

```javascript
export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:3001',
        changeOrigin: true
      }
    }
  },
  // 生产环境代理（Vercel 会处理）
})
```

## 五、快速部署脚本

### 一键部署到 Railway + Vercel

1. **推送代码到 GitHub**
```bash
git add .
git commit -m "feat: 喵健康 v1.0"
git push origin main
```

2. **在 Railway 部署后端**
   - 连接仓库
   - 选择 server 目录
   - 添加环境变量

3. **在 Vercel 部署前端**
   - 连接仓库
   - 选择 client 目录
   - 添加后端地址环境变量

## 七、常见问题

### Q: 前端请求后端跨域怎么办？
A: 后端已配置 CORS，允许所有来源访问。

### Q: 数据会丢失吗？
A: 当前使用内存存储，服务重启会丢失。建议：
- 升级使用 Railway PostgreSQL 数据库
- 或使用 MongoDB Atlas（免费 512MB）

### Q: 如何绑定自定义域名？
A: 
- Vercel：项目设置 → Domains → 添加域名
- Railway：项目设置 → Domains → 添加域名

## 八、升级建议

### 添加持久化数据库

1. 在 Railway 创建 PostgreSQL 数据库
2. 修改 `server/src/data/database.js` 使用真实数据库
3. 添加用户系统支持多用户

### 添加用户系统

1. 使用 JWT 认证
2. 每个用户只能看到自己的猫咪数据
3. 支持微信/邮箱登录
